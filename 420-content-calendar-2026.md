## OVERVIEW & STRATEGY

**Core content angle:** Price transparency. Every post leads with data, not hype.
The site's edge is the daily price gap across 20 licensed NYC dispensaries.
That gap is the story. 4/20 is just the hook to make people care about it.

**Platform priority:**
- Twitter/X: Primary distribution. Price data posts, daily cadence.
- Instagram: 3-4 posts total, lifestyle angle, countdown framing.
- Email: One pre-420 broadcast (send April 17 or 18).

**Tone:** Informative. Slightly dry. No dispensary cheerleading.
The site is a price tracker, not a marketing arm for any store.

**Placeholder data:** Posts below use [STORE] and [$X] brackets.
Fill in the actual scraped numbers from that morning's run before scheduling.
The structure of each post is the real asset — the data slots in.

## PRODUCTION NOTES

**Daily workflow to support this calendar:**
1. Scraper runs at 6 AM (already scheduled via Task Scheduler)
2. Pull that day's cheapest/most expensive from the output JSON or the live site
3. Drop numbers into the day's tweet template
4. Schedule or post

**The two numbers you need every morning:**
- Cheapest eighth: store name + price
- Most expensive eighth: store name + price
- Gap: (most expensive) minus (cheapest)

That's all you need to fill in every template in this calendar.

**If the scraper fails:**
Don't post fake data. Either skip the day or post a
"scraper is catching up" note. Credibility is the asset.

**Buffer/scheduling:** If you're using Buffer, Typefully, or
similar for Twitter, you can pre-load the template text and
fill in numbers each morning before the scheduled send time.

**Email platform note:** The email pipeline requires RESEND_API_KEY
to be set in the Railway environment. Set this before April 17
or you won't be able to send the broadcast.
